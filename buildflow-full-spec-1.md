# 1. EXECUTIVE SUMMARY

## 1.1 Product Vision

BuildFlow is a mobile-first construction scheduling and management platform designed specifically for residential home builder superintendents. It transforms chaotic spreadsheet-based scheduling into an intelligent, automated system that understands construction sequencing, parallel work tracks, and cascade effects.

## 1.2 Target Market

| Segment | Description | Size Estimate |
|---------|-------------|---------------|
| Production Builders | 50+ homes/year, multiple communities | ~2,500 companies |
| Regional Builders | 10-50 homes/year, 1-3 communities | ~15,000 companies |
| Custom Builders | <10 homes/year, high-touch | ~50,000 companies |

**Primary Target:** Production and Regional Builders (high volume, most pain)

## 1.3 Core Problems Solved

| Problem | Current State | BuildFlow Solution |
|---------|---------------|-------------------|
| Schedule Chaos | Excel spreadsheets, manual updates | Intelligent auto-scheduling |
| Cascade Confusion | Manual tracking of delays | Auto-cascade with parallel track awareness |
| Communication Gaps | Phone calls, texts scattered | Centralized messaging with templates |
| Weather Blindness | Reactive to weather | Proactive 7-day forecast integration |
| Reporting Burden | Hours building reports | One-click export, auto-reports |
| Inspection Tracking | Paper checklists, memory | Digital workflow with blocking logic |
| Accountability | "He said, she said" | Photo documentation, timestamps |

## 1.4 Key Differentiators

1. **Parallel Track Intelligence** - Understands that interior and exterior work are independent
2. **Smart Cascade Logic** - Only shifts tasks that actually depend on delayed work
3. **Mobile-First Design** - Built for job site use with offline capability
4. **Weather Integration** - Proactive alerts for outdoor tasks
5. **Superintendent-Centric** - Not accounting software with scheduling bolted on

## 1.5 Success Metrics

| Metric | Target | Measurement |
|--------|--------|-------------|
| Build Time Reduction | 10% improvement | Avg days to completion |
| On-Time Delivery | 85%+ | Lots completed by target date |
| Delay Visibility | 100% tracked | All delays logged with reason |
| Sub Communication | <5 min response | Avg time to notify affected subs |
| Report Generation | <60 seconds | Time from request to export |
| User Adoption | 80%+ daily active | Supers using app daily |

```typescript
  dependencies: Dependency[];

  status: TaskStatus;
  
  // Delay Info
  delay_days: number;                     // 0 if no delay
  delay_reason: DelayReason | null;
  delay_notes: string | null;
  delay_logged_at: DateTime | null;
  delay_logged_by: UUID | null;
  
  // Inspection
  requires_inspection: boolean;
  inspection_type: InspectionType | null;
  inspection_id: UUID | null;             // Link to inspection record
  
  // Flags
  is_outdoor: boolean;                    // Weather-sensitive
  is_critical_path: boolean;
  lead_time_days: number;                 // Material order lead time
  
  // Documentation
  photos: Photo[];
  notes: Note[];
  
  // Metadata
  sort_order: number;
  created_at: DateTime;
  updated_at: DateTime;
}

type TaskStatus = 
  | 'pending'                             // Not yet startable
  | 'ready'                               // Dependencies met, can start
  | 'in_progress'                         // Work underway
  | 'delayed'                             // Delayed, logged reason
  | 'blocked'                             // Waiting on inspection/other
  | 'complete';                           // Done

type Track = 
  | 'foundation'                          // Sequential, start of project
  | 'structure'                           // Sequential, framing/roof
  | 'interior'                            // Can run parallel with exterior
  | 'exterior'                            // Can run parallel with interior
  | 'final';                              // Sequential, end of project

type Phase = 
  | 'pre_construction'
  | 'foundation'
  | 'framing'
  | 'mechanical'
  | 'insulation_drywall'
  | 'finishes'
  | 'exterior'
  | 'final';

  | 'labor_no_show'
  | 'labor_shortage'
  | 'inspection_failed'
  | 'inspection_delayed'
  | 'permit_issue'
  | 'prior_trade'
  | 'change_order'
  | 'owner_request'
  | 'other';

  depends_on_task_id: UUID;
  type: DependencyType;
```

```sql
    first_name VARCHAR(100),

    phone VARCHAR(50),
    role VARCHAR(50) NOT NULL DEFAULT 'superintendent',

);
```

## 5.3 Schedule Engine

### 5.3.1 Schedule Generation Algorithm

**User Story:**
As a superintendent, when I start a lot, the system should automatically generate a complete schedule based on my template, skipping weekends and holidays.

**Algorithm Pseudocode:**

```typescript
function generateSchedule(
  startDate: Date,
  template: Template,
  orgSettings: OrgSettings
): Task[] {
  
  const schedule: Task[] = [];
  const taskEndDates: Map<UUID, Date> = new Map();
  const holidays = orgSettings.holidays;
  const workDays = orgSettings.work_days; // [1,2,3,4,5] = Mon-Fri
  
  // Sort template tasks by sort_order
  const sortedTasks = template.tasks.sort((a, b) => a.sort_order - b.sort_order);
  
  // Categorize tasks by track
  const foundationTasks = sortedTasks.filter(t => t.track === 'foundation');
  const structureTasks = sortedTasks.filter(t => t.track === 'structure');
  const interiorTasks = sortedTasks.filter(t => t.track === 'interior');
  const exteriorTasks = sortedTasks.filter(t => t.track === 'exterior');
  const finalTasks = sortedTasks.filter(t => t.track === 'final');
  
  // 1. Schedule Foundation (sequential)
  let currentDate = getNextWorkDay(startDate, holidays, workDays);
  for (const task of foundationTasks) {
    const taskStart = currentDate;
    const taskEnd = addWorkDays(taskStart, task.duration - 1, holidays, workDays);
    
    schedule.push({
      ...task,
      scheduled_start: taskStart,
      scheduled_end: taskEnd,
    });
    
    taskEndDates.set(task.id, taskEnd);
    currentDate = addWorkDays(taskEnd, 1, holidays, workDays);
  }
  
  // 2. Schedule Structure (sequential)
  for (const task of structureTasks) {
    const taskStart = currentDate;
    const taskEnd = addWorkDays(taskStart, task.duration - 1, holidays, workDays);
    
    schedule.push({
      ...task,
      scheduled_start: taskStart,
      scheduled_end: taskEnd,
    });
    
    taskEndDates.set(task.id, taskEnd);
    currentDate = addWorkDays(taskEnd, 1, holidays, workDays);
  }
  
  // 3. Find "Dried-In" point (after roofing)
  const roofingTask = schedule.find(t => t.name.includes('Roof'));
  const driedInDate = roofingTask ? taskEndDates.get(roofingTask.id) : currentDate;
  
  // 4. Schedule Interior Track (sequential within track)
  let interiorDate = addWorkDays(driedInDate, 1, holidays, workDays);
  for (const task of interiorTasks) {
    const taskStart = interiorDate;
    const taskEnd = addWorkDays(taskStart, task.duration - 1, holidays, workDays);
    
    schedule.push({
      ...task,
      scheduled_start: taskStart,
      scheduled_end: taskEnd,
    });
    
    taskEndDates.set(task.id, taskEnd);
    interiorDate = addWorkDays(taskEnd, 1, holidays, workDays);
  }
  
  // 5. Schedule Exterior Track (parallel with interior)
  // Can start after dried-in, runs independently
  let exteriorDate = addWorkDays(driedInDate, 1, holidays, workDays);
  for (const task of exteriorTasks) {
    const taskStart = exteriorDate;
    const taskEnd = addWorkDays(taskStart, task.duration - 1, holidays, workDays);
    
    schedule.push({
      ...task,
      scheduled_start: taskStart,
      scheduled_end: taskEnd,
    });
    
    taskEndDates.set(task.id, taskEnd);
    exteriorDate = addWorkDays(taskEnd, 1, holidays, workDays);
  }
  
  // 6. Schedule Final Track (must wait for BOTH interior and exterior)
  const interiorEndDate = Math.max(
    ...interiorTasks.map(t => taskEndDates.get(t.id).getTime())
  );
  const exteriorEndDate = Math.max(
    ...exteriorTasks.map(t => taskEndDates.get(t.id).getTime())
  );
  
  let finalDate = addWorkDays(
    new Date(Math.max(interiorEndDate, exteriorEndDate)),
    1,
    holidays,
    workDays
  );
  
  for (const task of finalTasks) {
    const taskStart = finalDate;
    const taskEnd = addWorkDays(taskStart, task.duration - 1, holidays, workDays);
    
    schedule.push({
      ...task,
      scheduled_start: taskStart,
      scheduled_end: taskEnd,
    });
    
    finalDate = addWorkDays(taskEnd, 1, holidays, workDays);
  }
  
  // 7. Assign preferred subs to each task
  for (const task of schedule) {
    task.sub_id = getPreferredSubForTrade(task.trade);
  }
  
  // 8. Calculate critical path
  markCriticalPathTasks(schedule);
  
  return schedule;
}

// Helper functions
function getNextWorkDay(date: Date, holidays: Holiday[], workDays: number[]): Date {
  let d = new Date(date);
  while (!isWorkDay(d, holidays, workDays)) {
    d.setDate(d.getDate() + 1);
  }
  return d;
}

function addWorkDays(
  date: Date, 
  days: number, 
  holidays: Holiday[], 
  workDays: number[]
): Date {
  let d = new Date(date);
  let added = 0;
  while (added < days) {
    d.setDate(d.getDate() + 1);
    if (isWorkDay(d, holidays, workDays)) {
      added++;
    }
  }
  return d;
}

function isWorkDay(date: Date, holidays: Holiday[], workDays: number[]): boolean {
  const dayOfWeek = date.getDay(); // 0 = Sunday, 1 = Monday, etc.
  const isHoliday = holidays.some(h => isSameDay(new Date(h.date), date));
  return workDays.includes(dayOfWeek) && !isHoliday;
}
```

### 5.4.3 Cascade Logic

**When a task is delayed:**

```typescript
function applyDelay(
  schedule: Task[],
  delayedTaskId: UUID,
  delayDays: number,
  reason: DelayReason
): Task[] {
  
  const taskIndex = schedule.findIndex(t => t.id === delayedTaskId);
  const delayedTask = schedule[taskIndex];
  const delayedTrack = delayedTask.track;
  
  // Update the delayed task
  delayedTask.delay_days = delayDays;
  delayedTask.delay_reason = reason;
  delayedTask.status = 'delayed';
  
  // Shift dates for delayed task
  delayedTask.scheduled_start = addWorkDays(delayedTask.scheduled_start, delayDays);
  delayedTask.scheduled_end = addWorkDays(delayedTask.scheduled_end, delayDays);
  
  // Find all dependent tasks
  for (let i = taskIndex + 1; i < schedule.length; i++) {
    const task = schedule[i];
    
    // Check if this task depends on the delayed task
    const dependsOnDelayed = task.dependencies.some(
      d => d.depends_on_task_id === delayedTaskId
    );
    
    // Check if same track (must cascade)
    const sameTrack = task.track === delayedTrack;
    
    // Check if this is final phase (waits for all)
    const isFinalPhase = task.track === 'final';
    
    if (dependsOnDelayed || sameTrack) {
      // Cascade the delay
      task.scheduled_start = addWorkDays(task.scheduled_start, delayDays);
      task.scheduled_end = addWorkDays(task.scheduled_end, delayDays);
    } else if (isFinalPhase) {
      // Final phase only shifts if the delayed track is now the bottleneck
      const interiorEnd = getTrackEndDate(schedule, 'interior');
      const exteriorEnd = getTrackEndDate(schedule, 'exterior');
      const newFinalStart = addWorkDays(
        new Date(Math.max(interiorEnd, exteriorEnd)),
        1
      );
      
      if (newFinalStart > task.scheduled_start) {
        const shift = daysBetween(task.scheduled_start, newFinalStart);
        task.scheduled_start = newFinalStart;
        task.scheduled_end = addWorkDays(task.scheduled_end, shift);
      }
    }
    // IMPORTANT: Tasks on OTHER track do NOT cascade
  }
  
  return schedule;
}
```

**Example Scenario:**

```

Interior: Drywall (Day 40-48) â†’ Paint (Day 49-53)
Exterior: Siding (Day 40-44) â†’ Gutters (Day 45)

Drywall delayed 5 days:

AFTER DELAY:
Interior: Drywall (Day 45-53) â†’ Paint (Day 54-58) [SHIFTED]
Exterior: Siding (Day 40-44) â†’ Gutters (Day 45)  [NOT SHIFTED]

Final Phase: Must now wait for Interior (Day 58) instead of Exterior (Day 45)
```

### 5.6.3 Calendar Drag-and-Drop

**Rules:**
- Cannot violate hard dependencies (system warns)
- Can stack tasks on same day if different tracks
- Shows "what if" preview before confirming

**Drag Behavior:**

```
1. User taps and holds task bar
2. Task lifts with shadow effect
3. User drags to new date
4. System shows:
   - Green highlight = valid drop
   - Red highlight = dependency violation
   - Yellow highlight = sub conflict
5. On drop:
   - If valid: Show confirmation modal
   - If invalid: Show error and snap back
```
